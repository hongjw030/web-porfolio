코드잇 프론트엔드 부트캠프 수료

코드잇 주최

개발자는 많은 사람들과 소통하고 협력하는 것이 중요함을 깨닫고, 저와 같은 꿈을 가진 열정적인 사람들을 만나 네트워킹을 쌓기 위해 부트캠프를 수료했습니다. 이 과정에서 프론트엔드 실무에 대해 공부하고, 협업 프로젝트를 진행했습니다.

Skills

Language

Javascript의 기본적인 문법 및 동적 타이핑 특징을 이해하고, 브라우저에서의 Javascript 동작 원리와 순서를 공부했습니다.
동적 타입 언어인 Javascript는 대규모 프로젝트에서 에러가 발생할 가능성이 커진다는 점을 인지하고, 그의 슈퍼셋인 Typescript 언어를 선택해 주력으로 공부하고 있습니다. '모던 타입스크립트 딥 다이브' 등의 책으로 이론을 공부하고, 타입스크립트가 어떤 특징을 가지고 있는지 숙지하며 개발하려 노력합니다.

DB

NoSQL문을 사용해 데이터베이스를 구현할 수 있습니다. mongoDB는 소규모 프로젝트에서 무료로 데이터를 저장하기 쉽고, vercel과 연동하기 편하다는 장점이 있어 선택했습니다. 데이터베이스를 설계할 때 적절한 정규화를 선택해 구현하려 합니다.

Framework & Library

프론트엔드 라이브러리 중 커뮤니티가 크고, 다양한 외부 라이브러리와 프레임워크와 호환된다는 장점이 있어 선택했습니다. React JS 18의 기본적인 사용법을 숙지하고,브라우저에서 가상돔이 어떻게 작동하는지, 훅이 어떻게 작동하는지를 공부하여 더욱 최적화된 웹을 만들기 위해 노력합니다.
next/font, next/Image 등 내장된 최적화 컴포넌트가 제공된다는 점, SSR과 SSG를 활용할 수 있단 점에서 해당 프레임워크를 선택했습니다. 또한 api routes로 api 엔드포인트를 설정하기 편하다는 장점이 있어 선택했습니다. Next.js의 pages routing으로 프로젝트를 개발한 경험이 있으며, 현재는 app routing을 공부 중입니다.
api 횟수를 줄이고 적절한 캐싱을 활용하는 것이 웹 성능에 큰 영향을 미침을 깨닫고, 서버 상태관리 라이브러리인 React Query를 공부했습니다. 해당 라이브러리로 무한 스크롤, 페이지네이션을 구현한 경험이 있습니다.
React에서 제공하는 Context api의 불필요한 렌더링 유발 문제로 인해 외부 상태관리 라이브러리의 필요성을 깨닫고 Jotai를 선택했습니다.

Design

제한된 시간 내에 빠르게 프로젝트를 완성해야 할 때, css에 시간을 덜 들이고자 tailwind css를 선택해 사용한 경험이 있습니다.
scss, sass 문법을 공부해 프로젝트에 적용한 경험이 있습니다. 일반 css와 문법이 비슷해 러닝커브가 적으면서 네스팅 문법, 연산자 등을 활용해 더 가독성 있는 코드를 짤 수 있다는 장점이 있어 해당 라이브러리를 공부했습니다.
스타일을 분리하지 않고 하나의 컴포넌트 레벨로 간주하기 떄문에, React와 잘 조합되는 장점이 있어 공부했습니다.
1인 프로젝트를 구현할 때 디자인 시스템에 드는 공수를 줄이고자 반 완제품을 제공하는 디자인 라이브러리, MUI를 선택해 공부했습니다. 커뮤니티가 크고 이미 완성된 형태의 컴포넌트를 제공한다는 장점이 있어, 해당 라이브러리를 사용했습니다. 덕분에 프로젝트 디자인에 드는 시간을 80%이상 줄인 경험이 있습니다.
디자인 라이브러리 MUI는 이미 정형화된 컴포넌트로 디자인에 신경쓰지 않아도 된다는 장점이 있지만, 자유로운 커스텀이 제한됐다는 단점이 있었습니다. 이에, 유연한 커스텀 및 테마 적용을 위해 Chakra UI를 선택해 마이그레이션 한 경험이 있습니다.

Communication

Git의 분산 버전 관리 시스템 특성을 숙지해 활용하고 있습니다. PR, Issue, 코드 리뷰로 팀원과 소통할 수 있고, Git Flow 전략을 택해 적절한 브랜치를 생성할 수 있습니다. 이 외에도 Github의 정적 페이지 호스팅, Git Project와 마일스톤을 활용한 일정 관리를 진행할 수 있습니다.
Trello를 활용해 팀원들과 칸반 개발 방법론을 활용한 경험이 있습니다. 소규모 프로젝트였던 만큼 러닝 커브가 적고 간편한 툴을 사용하고자 Trello를 택했습니다.
데이터베이스로서의 Notion을 활용할 수 있습니다.
